02-03 SHiCI Response Specification



1. Purpose

Response Specificationは、SHiCIが構築したKnowledgeを、人間および外部システムへどのように表現するかを定義する設計仕様である。

Response Specificationは表示方法を定義するものであり、Knowledgeそのものを定義するものではない。

Knowledgeの構築はProcessing ModelのComposeで行われる。

Response Specificationは、そのKnowledgeをExpressし、Outputする方法を定義する。



2. Scope

Response Specificationは以下を対象とする。

・自然言語による回答

・Entity Card

・Snapshot表示

・一覧表示

・比較表示

・Timeline表示

・Table表示

・JSON出力

・API Response

Response Specificationは推論を定義しない。

Response SpecificationはKnowledgeを生成しない。



3. Responsibility

Response Engineの責務は以下に限定する。

3.1 表現

Knowledgeを最適な形式へ変換する。

3.2 一貫性

同じKnowledgeは常に同じ情報を返す。

表現形式が異なっても意味は変化しない。

3.3 可読性

利用者にとって理解しやすいResponseを生成する。

Response Engineは推論を行わない。

Response EngineはKnowledgeを書き換えない。

6. Response Principles

Responseは以下の原則に従う。

Knowledge First

ResponseはKnowledgeを表現する。

Responseの都合でKnowledgeを変更してはならない。

Human First

人間が理解しやすいResponseを優先する。

AI Friendly

AIが利用しやすい構造も維持する。

Single Meaning

一つのKnowledgeは一つの意味しか持たない。

表現形式が変わっても意味は変化しない。

Progressive Disclosure

必要な情報だけを表示する。

詳細は必要になった時点で表示する。



7. Response Composition

ResponseはCompose済みKnowledgeから構築する。

Responseは新しいKnowledgeを生成しない。

ResponseはCompose結果を整形するだけである。



8. Design Principles

Response Engineは以下の原則に従う。

Presentation Only

表示のみを担当する。

Stateless

Responseは状態を保持しない。

Conversation StateはConversationStateEngineが管理する。

Entity Centered

Entityを中心に表示する。

Snapshot Oriented

Snapshotを表示単位とする。

Consistent

同じEntityは常に同じ構成で表示する。

Conclusion

Response Specificationは、Knowledgeを利用者へ正しく届けるための設計仕様である。

ResponseはKnowledgeを生成しない。

ResponseはKnowledgeを変更しない。

ResponseはKnowledgeを最適な形式で表現することのみを責務とする。

SHiCIはKnowledgeを中心とした設計を維持し、ResponseはそのKnowledgeを利用者へ届けるための最終工程として機能する。
