02-99 SHiCI Architecture Map



1. Purpose

Architecture Mapは、SHiCIの設計仕様と実装コードとの対応関係を定義する文書である。

本書は、設計書をコードへ正しく反映し、設計と実装の一貫性を維持することを目的とする。

Architecture Mapはシステム仕様を定義するものではない。

Architecture Mapは設計書と実装コードを結び付けるための対応表である。




2. Scope

Architecture Mapは以下を対象とする。

・設計書

・エンジン

・Handler

・Snapshot

・Response

・Frontend

Architecture Mapはデータ構造を定義しない。

Architecture Mapはアルゴリズムを定義しない。

8. Future Refactoring

AnswerBuilderは将来的に廃止する。

Express層はResponse Specificationへ統合する。

Conversation Stateは正式ルートへ統合する。

HandlerはEntity処理のみ担当する。





9. Design Principles

Architecture Mapは設計書を置き換えない。

Architecture Mapはコードを置き換えない。

Architecture Mapは設計とコードを結び付ける。

Architecture Mapは実装変更時に必ず更新する。





10. Maintenance Rules

新しいEngineを追加した場合。

Architecture Mapを更新する。

新しい設計書を追加した場合。

Architecture Mapを更新する。

責務変更を行った場合。

Architecture Mapを更新する。

Conclusion

Architecture Mapは、SHiCIにおける設計書と実装コードとの対応関係を定義する文書である。

Architecture Mapはシステム仕様ではない。

Architecture Mapは設計と実装の整合性を維持し、開発者が設計思想をコードへ正しく反映するための指針となる。
